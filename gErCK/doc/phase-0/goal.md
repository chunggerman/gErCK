# Phase 0 — Goals
On Your Left — gErCK

Phase 0 focuses on clarity, structure, and readiness.

## Primary Goals

### 1. A Clean, Modular Codebase
The repo should feel:
- organized
- predictable
- scalable
- easy to navigate

### 2. A Working Backend Skeleton
Includes:
- FastAPI app
- routers
- capability orchestrator
- function layer
- LLM gateway
- storage abstraction

### 3. A Minimal Frontend
Includes:
- basic Next.js app
- simple UI
- file upload
- API call example

### 4. A Unified Docker Environment
- backend container
- frontend container
- shared volume
- environment variables

### 5. CI/CD Pipeline
- linting
- tests
- build pipeline

### 6. Security & Governance Baseline
- security checklist
- pentest readiness
- logging baseline

### 7. Documentation
- architecture
- folder structure
- backend foundation
- frontend foundation
- docker environment
- CI/CD
- test plan
